## 摘要
检索增强生成（RAG）系统通常将文档视为扁平文本，忽略了知识图谱提供的结构化元数据和链接关系。在本文中，我们研究了结构化链接数据（特别是 Schema.org 标记和由链接数据平台提供的可解引用实体页面）是否能提高标准 RAG 和智能体 RAG 系统的检索准确率和回答质量。我们在四个领域（编辑、法律、旅游、电子商务）进行了对照实验，使用 Vertex AI Vector Search 2.0 进行检索，并使用 Google Agent Development Kit (ADK) 进行智能体推理。我们的实验设计测试了七种情况：三种文档表示形式（纯 HTML、带有 JSON-LD 的 HTML 以及增强的智能体优化实体页面）与两种检索模式（标准 RAG 和具有多跳链接遍历的智能体 RAG）交叉，外加一种增加了丰富导航功能和实体互联的 Enhanced+ 情况。我们的结果表明，虽然仅靠 JSON-LD 标记只能带来适度的改进，但我们结合了 llms.txt 风格的智能体指令、面包屑导航和神经搜索能力的增强实体页面格式实现了显著提升：标准 RAG 的准确率提高了 29.6%，完整智能体流水线的准确率提高了 29.8%。具有更丰富导航功能的 Enhanced+ 变体获得了最高的绝对分数（准确率：4.85/5，完整性：4.55/5），尽管其相对于基础增强格式的增量收益在统计上并不显著。我们发布了数据集、评估框架和增强的实体页面模板，以支持可复现性。

## Abstract
Retrieval-Augmented Generation (RAG) systems typically treat documents as flat text, ignoring the structured metadata and linked relationships that knowledge graphs provide. In this paper, we investigate whether structured linked data, specifically Schema.org markup and dereferenceable entity pages served by a Linked Data Platform, can improve retrieval accuracy and answer quality in both standard and agentic RAG systems. We conduct a controlled experiment across four domains (editorial, legal, travel, e-commerce) using Vertex AI Vector Search 2.0 for retrieval and the Google Agent Development Kit (ADK) for agentic reasoning. Our experimental design tests seven conditions: three document representations (plain HTML, HTML with JSON-LD, and an enhanced agentic-optimized entity page) crossed with two retrieval modes (standard RAG and agentic RAG with multi-hop link traversal), plus an Enhanced+ condition that adds rich navigational affordances and entity interlinking. Our results reveal that while JSON-LD markup alone provides only modest improvements, our enhanced entity page format, incorporating llms.txt-style agent instructions, breadcrumbs, and neural search capabilities, achieves substantial gains: +29.6% accuracy improvement for standard RAG and +29.8% for the full agentic pipeline. The Enhanced+ variant, with richer navigational affordances, achieves the highest absolute scores (accuracy: 4.85/5, completeness: 4.55/5), though the incremental gain over the base enhanced format is not statistically significant. We release our dataset, evaluation framework, and enhanced entity page templates to support reproducibility.

传统的 RAG 系统通常将网页或文档视为**扁平的无结构文本**，在向量化过程中忽略了网页中广泛存在的结构化元数据（如 Schema.org 标记）和实体间的链接关系。随着 AI 驱动的搜索引擎（如 Google AI Mode）兴起，如何让 AI 智能体更好地理解和遍历互联网上的知识图谱变得至关重要。本研究旨在验证：**结构化链接数据能否提高 RAG 的准确性，以及智能体化的链接遍历（Link Traversal）能否带来额外增益。**

### 2. 方法论：核心思想与技术细节
作者提出了一种将结构化数据转化为“智能体友好型”文档的方案，核心组件包括：
*   **增强型实体页面（Enhanced Entity Page）：** 这是一种专为 AI 智能体设计的文档格式，包含：
    *   基于结构化数据生成的自然语言摘要。
    *   嵌入式 JSON-LD 块（提供机器可读的精确属性）。
    *   **llms.txt 风格指令：** 为 LLM 提供显式的操作指南。
    *   **可解引用的 URI 导航：** 将实体间的关系显式化，允许智能体通过 HTTP 请求获取关联实体的详细数据。
*   **智能体编排检索（Agent-Orchestrated Retrieval）：**
    *   使用 **Google ADK** 构建具备 ReAct 推理环路的智能体。
    *   **工具集：** 包含向量搜索（Vertex AI）、链接跟随（解引用 URI 获取更多上下文）和知识图谱神经搜索。
    *   **多跳推理：** 智能体可以根据初始检索结果中的链接，自主决定是否跳转到相关实体页面以补充信息。

### 6. 主要结论与发现
*   **JSON-LD 效果有限：** 在扁平文本检索架构中，仅简单添加 JSON-LD 标记对准确率的提升非常微小（仅 +0.17）。
*   **增强型页面效果显著：** 采用“增强型实体页面”后，标准 RAG 的准确率提升了 **29.6%**，智能体流水线的准确率提升了 **29.8%**。
*   **智能体的补偿作用：** 智能体在处理结构化程度较差的文档时提升最大（+0.74）；但如果文档本身已经过“增强型”优化，智能体带来的额外准确率增益会边际递减，其主要价值转为提高检索效率（减少工具调用次数）。
*   **链接物化（Link Materialization）：** 将原本隐藏在 URI 后的关联数据转化为自然语言描述，是提升 LLM 理解能力的关键。

### 7. 优点（亮点）
*   **工程实用性：** 提出了一种切实可行的“SEO 3.0”优化策略，即通过优化页面结构来迎合 AI 智能体的检索偏好。
*   **弥合鸿沟：** 成功将语义网（Semantic Web）的链接数据理念与现代大模型智能体架构相结合。
*   **开源贡献：** 发布了数据集、评估框架和模板，具有良好的可复现性。

### 8. 不足与局限
*   **信息量与格式的混淆：** 增强型页面在改变格式的同时也引入了更多关联实体的信息，实验未能完全隔离“格式优化”与“信息增量”各自的贡献度。
*   **向量数据库限制：** 实验中使用的 Vertex AI 对文档长度有截断限制，可能导致部分 JSON-LD 数据在索引时丢失，这可能低估了纯 JSON-LD 的潜力。
*   **模型家族偏差：** 生成、推理和评估均使用了 Gemini 系列模型，可能存在家族内部的偏好一致性风险。
*   **知识图谱依赖：** 该方法高度依赖于高质量、维护良好的知识图谱底层数据，对于数据质量差的领域可能效果受限。
